# Notifications & Deploy

## Notifications

The notification bell in the header keeps you informed of important events.

### Notification Types

| Type | Color | Examples |
|------|-------|----------|
| **Info** | Blue | Session started, status changed |
| **Success** | Green | PR merged, verification passed |
| **Warning** | Yellow | Agent escalation, review needed |
| **Error** | Red | Session failed, CI failure |

### Managing Notifications

- Click the **bell icon** to open the notification panel
- Click a notification to mark it as read
- Use **Mark all read** to clear the unread badge
- Use **Clear all** to remove all notifications
- Some notifications have **action buttons** (e.g., "Open Settings")

### Notification Sources

Notifications are generated by:
- TicketAgent status changes and escalations
- Claude Code session events
- CI/CD pipeline results
- GitHub webhook events

## Deploy Status

The deploy status widget in the header shows the current state of your CI/CD pipeline.

### Status Indicators

- **Green checkmark** — latest deploy succeeded
- **Yellow spinner** — deploy in progress
- **Red X** — deploy failed

### Workflow Runs

Click the deploy widget to see recent GitHub Actions workflow runs with:
- Run status and conclusion
- Trigger event (push, PR, manual)
- Duration and timestamp
- Link to the run on GitHub
